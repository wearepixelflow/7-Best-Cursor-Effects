lerp.js
// requestAnimationFrame loop
function animate() {
ringX += (mouseX - ringX) * 0.15;
ringY += (mouseY - ringY) * 0.15;
moveRing(ringX, ringY);
requestAnimationFrame(animate);
}
How it works
The dot is your cursor
Updated synchronously inside mousemove. No rAF, no math. Always at exactly (x, y).
The ring uses linear interpolation
Each frame: current += (target - current) × 0.15. This means it always closes 15% of the remaining gap — fast at first, then slows exponentially.
Why 0.15 feels premium
Too high (0.5+) and the ring snaps. Too low (0.05) and it floats sluggishly. 0.15 is the sweet spot: visible lag, satisfying catch-up, buttery feel.
